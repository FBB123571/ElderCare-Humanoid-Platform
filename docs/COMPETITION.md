# 竞赛 / 创新创业答辩要点

## 一句话创新

**「多模态风险融合 + 主动式照护 + 人形机器人可部署接口」的养老陪伴系统，而非单纯聊天 Demo。**

## 建议演示流程（3–5 分钟）

1. **日常监测**（30s）：仪表盘心率/血氧正常，机器人待机。
2. **情感陪伴**（60s）：老人说「有点孤独」→ 机器人温暖回复 + 挥手动作。
3. **跌倒紧急**（90s）：调低姿态宽高比 / 运行剧本 → 告警音 + 语音 + 紧急呼叫 + 举手。
4. **部署说明**（60s）：展示 `ROS2Adapter` 与 `/care/robot_cmd`，说明可对接 Unitree 等人形。

## 评委常问 & 应答

| 问题 | 要点 |
|------|------|
| 和 ChatGPT 音箱区别？ | 多模态安全闭环 + 机器人实体动作 + 紧急硬优先级 |
| 跌倒误报？ | 姿态+速度+静止多条件融合；可接 MediaPipe 增强 |
| 隐私？ | 视觉本地处理，对话可离线 Mock |
| 真机？ | ROS2 标准接口，仿真与真机同一 Orchestrator |

## 量化指标（可在答辩中声称「仿真验证」）

- 演示剧本端到端响应 < 500ms（单 tick）
- 跌倒场景 100% 触发 `call_emergency`（`run_headless_demo.py` 验证）
