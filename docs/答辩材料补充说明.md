# 答辩材料补充说明（队内自用）

## 目前已齐

- 答辩 PPT：`docs/答辩_CareCompanion.pptx`（含 Web 界面、跌倒紧急、MediaPipe 全身骨架图）
- 技术报告：`docs/TECHNICAL_REPORT.md`
- 评测数据：`data/evaluation/results/fall_eval_report.json`
- 开源仓库：GitHub 可克隆、可跑 Web 与无头剧本
- 现场演示：`bash scripts/run_web.sh` → `http://localhost:8765`

## 建议继续补的（按优先级）

### 1. 演示视频（3～5 分钟）

用手机或 OBS 录屏即可，顺序建议：

1. 打开 Web 控制台
2. 上传照片 → 出现骨架（你已有全身图，可直接用「保存骨架图」导出高清帧插入视频）
3. 老人倾诉场景 → 风险面板
4. 模拟跌倒 → 紧急呼叫与指令流

导出后上传 B 站（未列出）或网盘，PPT 最后一页放二维码。

### 2. 封面信息

在 PPT 封面手写或改字：

- 学校、队伍名
- 队员、指导教师
- 第二十八届国赛 · 机器人创新赛道

### 3. 软著与知识产权

若已申请：把登记号填进 PPT「开源与知识产权」页；  
若未申请：写「软件著作权申请中」，附系统截图与代码目录打印件备查。

### 4. 真机或仿真（加分项）

- Unitree / 其他 ROS2 人形：拍 30 秒「语音 + 举手」实拍  
- 暂无真机：用 `run_headless.sh` 日志 + 指令流截图说明「接口已打通」

### 5. 对比与数据（评委爱问）

准备一页口头说明即可，不必再写长文：

- 与手环/摄像头/音箱相比，我们多了**可执行动作**和**紧急硬优先级**
- 合成场景 F1=100% 的测试方法：`scripts/eval_fall_detection.py`，可现场重跑

### 6. 现场备份

- 笔记本本地已 `git pull`，`.venv` 与 `data/models/pose_landmarker_lite.task` 齐全
- 无网时用 `bash scripts/run_headless.sh` 兜底
- 骨架演示样图：`web/static/samples/pose_demo.jpg`（放一张备用图）

## 答辩话术注意

- 少说空泛形容词，多指到界面按钮和日志字段
- 视觉部分如实说：当前以**上传照片 + MediaPipe**为主，摄像头受距离与光线影响
- 跌倒检测交代清楚：合成场景评测通过，公开数据集与真机仍在后续计划里
