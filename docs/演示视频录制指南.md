# 演示视频录制指南（3～5 分钟）

## 零、服务器自动生成（不用自己录屏）

在 SSH 终端执行（需已安装 `playwright` 与 `ffmpeg`）：

```bash
cd /mnt/sdb1/leijh/EnergySnake1/robot/ElderCare-Humanoid-Platform
source .venv/bin/activate
pip install playwright
playwright install chromium
bash scripts/download_mediapipe_models.sh   # 若未下载
python scripts/record_demo_video.py
```

生成文件：`docs/assets/demo_carecompanion.mp4`（约 1MB，含上传骨架、老人倾诉、演示剧本、指令流）。

拷到本机：在 Cursor 文件树里右键该文件下载，或 `scp` 到 Windows。

## 一、录之前准备好（自己录屏时用）

### 1. 服务器上启动 Web

在 Cursor **底部 SSH 终端**执行：

```bash
cd /mnt/sdb1/leijh/EnergySnake1/robot/ElderCare-Humanoid-Platform
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY ALL_PROXY all_proxy
bash scripts/run_web.sh
```

看到 `Uvicorn running on http://0.0.0.0:8765` 即可。

### 2. 本机打开页面

1. Cursor 左侧 **「端口」** 面板找到 `8765`
2. 点 **「在浏览器中打开」**
3. 地址应是：`http://localhost:8765`
4. 浏览器按 **Ctrl+F5** 强刷，确认能看到「上传照片分析」「保存骨架图」

### 3. 准备一张照片

- 老人全身或上半身（你 PPT 里那张即可）
- 提前放在桌面，录的时候点「上传照片分析」选它

### 4. 录屏软件（Windows 任选一种）

| 方式 | 操作 |
|------|------|
| **Win + G**（Xbox 游戏栏） | 点录制按钮，录完在「视频/捕获」文件夹找 mp4 |
| **OBS Studio**（免费） | 添加「窗口捕获」选浏览器 → 开始录制 |
| **QQ / 微信** | 屏幕分享录屏（清晰度略低，应急可用） |

建议：**1920×1080、30fps**，只录浏览器窗口，不要录整个桌面（避免露出聊天框）。

## 二、推荐分镜脚本（总时长约 4 分钟）

按下面顺序录，**一段录完再录下一段**，后期可以剪在一起；也可以一镜到底。

| 时间 | 画面操作 | 口头讲解（可参考，不必背稿） |
|------|----------|------------------------------|
| **0:00～0:30** | 展示首页全貌，指一下标题 CareCompanion | 这是我们做的养老人形陪伴控制台，感知、决策、执行都在一个页面里。 |
| **0:30～1:20** | 点 **上传照片分析** → 选全身照 → 等骨架出来 → 指一下绿色点和线 → 可点 **保存骨架图** | 视觉用 MediaPipe 做姿态，上传照片就能出骨架和宽高比，后面跌倒检测会用到这些几何量。 |
| **1:20～2:10** | 点 **老人倾诉**（或选情绪「低落」）→ 输入「最近有点孤独，睡不太好」→ **单步推理** | 老人说话和情绪会进风险引擎，这里是陪伴场景，机器人会给安抚话术。 |
| **2:10～2:40** | 滚动右侧，指风险分数、机器人回应、四项指标 | 右边是融合后的风险面板，分数、原因、回复都能看到。 |
| **2:40～3:30** | 点 **模拟跌倒** 或 **运行演示剧本**（等剧本跑到跌倒） | 跌倒会拉高风险，状态切到紧急，下面日志里能看到告警和呼叫。 |
| **3:30～4:20** | 滚到 **交互日志**、**机器人指令流**（告警音→语音→紧急呼叫→手势） | 紧急时按固定顺序下发指令，家属推送在日志里可追溯。 |
| **4:20～4:40** | 可选：指一下 GitHub 仓库页或 README | 代码开源，评测脚本可以复现 F1 结果。 |

**注意：**

- 点按钮不要太快，等页面更新完再往下。
- 若剧本按钮一次跑完全程，可只录一条「运行演示剧本」，时间紧就 3 分钟版。
- 录坏了就重录该段，用 **剪映 / CapCut** 拼接。

## 三、视频导出与上传

### 导出

- OBS：停止录制 → 默认在「视频」文件夹，格式 mp4
- Win+G：设置里可看保存路径

检查：能正常播放、字看得清、时长 **3～5 分钟**。

### 上传到哪里

任选其一（答辩 PPT 里写清楚即可）：

| 平台 | 说明 |
|------|------|
| **哔哩哔哩** | 投稿 → 科技/生活区 → 标题如「CareCompanion 养老陪伴系统演示」→ 谁可以看选「公开」或「仅链接」 |
| **百度网盘** | 上传 mp4 → 分享 → 复制链接 |
| **GitHub Release** | 仓库 → Releases → 上传 mp4（文件别太大，<100MB 较合适） |

记下 **最终可打开的链接**，后面做二维码用。

## 四、做二维码放进 PPT

### 1. 生成二维码图片

1. 打开 https://cli.im/ （草料二维码，免费够用）
2. 选「文本」或「网址」
3. 粘贴你的视频链接（B 站视频页链接 / 网盘分享链接）
4. 下载 PNG，命名为：`docs/assets/ppt_demo_qr.png`

### 2. 插入 PPT

**方式 A（手动，推荐）**

1. 用 WPS / PowerPoint 打开 `docs/答辩_CareCompanion.pptx`
2. 在 **「敬请批评指正」** 前一页新建一页
3. 标题：**现场演示视频**
4. 插入图片 `ppt_demo_qr.png`，下面一行小字：「扫码观看系统演示」

**方式 B（重新生成 PPT，已放二维码文件时）**

把 `ppt_demo_qr.png` 放进 `docs/assets/` 后执行：

```bash
python tools/generate_presentation.py
```

（若仓库脚本已支持自动插入二维码页则会出现；否则用手动方式 A。）

### 3. 答辩时

- 投影 PPT 到二维码页，评委扫码即可看
- 现场仍建议 **笔记本直接开 Web** 演示，视频作备用

## 五、常见坑

| 现象 | 处理 |
|------|------|
| 录屏黑屏 | 换 OBS「窗口捕获」或浏览器窗口模式 |
| 网页打不开 | 检查 SSH 端口 8765 是否转发 |
| 上传没骨架 | 先 `bash scripts/download_mediapipe_models.sh`，重启 `run_web.sh` |
| 视频太大传不动 B 站 | 剪映导出时选 1080p、码率 5～8 Mbps |
| 二维码扫不开 | 用手机先自己扫一遍再放进 PPT |
