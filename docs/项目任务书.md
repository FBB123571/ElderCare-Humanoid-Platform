## 二、项目背景与意义

我国已进入深度老龄化阶段，居家老人跌倒、慢性不适与情感孤独等问题突出，传统固定式监护设备难以提供「感知—决策—执行—陪伴」一体化服务。人形机器人具备移动与操作潜力，但真机成本高、调试周期长，亟需可复现的仿真与软件平台支撑算法验证与答辩展示。

本项目 **CareCompanion** 面向居家养老场景，构建可运行的软件仿真平台：融合视觉跌倒检测、情感与健康风险、大模型对话及机器人动作编排，并通过 Web 控制台、数字人剧场与 Isaac Sim 双角色仿真视频，完整呈现「发现风险 → 决策下发 → 人形赶赴搀扶」的闭环能力。

## 三、任务目标

### 3.1 总体目标

建成一套 **可演示、可评测、可部署接口** 的智能养老陪伴机器人仿真平台，支撑国赛答辩与后续 ROS2 / 人形真机对接。

### 3.2 分项目标

| 编号 | 任务 | 验收标准 |
|------|------|----------|
| T1 | 多模态风险融合决策 | 跌倒 / 情绪 / 健康加权融合，输出 normal / alert / emergency |
| T2 | 视觉跌倒检测 | MediaPipe 骨架 + 规则/融合检测；合成场景 P/R/F1 达 100% |
| T3 | 情感陪伴与紧急响应 | Web 对话、模拟跌倒、紧急指令与通知链路 100% 触发 |
| T4 | 机器人执行接口 | 统一 RobotAction；ROS2 `/care/robot_cmd` 可 echo 演示 |
| T5 | 数字人情景演绎 | 依据视频心情轨迹生成老人—小护双角色对白短片 |
| T6 | 人形仿真执行层 | Isaac Sim：G1 + 老人同场景，跌倒—赶赴—搀扶站起录屏 |
| T7 | 答辩材料 | 技术报告、评测数据、嵌入视频的 PPT、约 47s 全流程总片 |

### 3.3 技术指标

## 四、系统总体设计

### 4.1 逻辑架构

```
多模态感知（视觉 / 对话 / 健康）
        ↓
风险融合引擎（Fall + Emotion + Health）
        ↓
照护编排 CareOrchestrator
        ↓
RobotAction → ROS2 / 仿真 / Web 展示
```

### 4.2 软件模块

| 模块 | 路径 | 职责 |
|------|------|------|
| 感知 | `care_companion/perception/` | 跌倒检测、姿态、视频分析 |
| 认知 | `care_companion/cognition/` | 对话、规划、数字人剧本 |
| 执行 | `care_companion/execution/` | 机器人适配、紧急通知 |
| Web | `web/` | FastAPI + 前端演示 |
| 工具链 | `scripts/`、`tools/` | 录屏、合成、PPT、评测 |

## 五、关键技术与创新点

1. **多模态可解释风险融合**：跌倒、情绪、健康统一评分，紧急态硬优先级覆盖。
2. **MediaPipe 本地视觉跌倒链**：支持上传视频/摄像头帧，骨架叠加与 FALL ALERT 可视化。
3. **心情驱动的数字人剧场**：由分析结果自动生成老人与小护对白脚本并渲染短片。
4. **决策与执行解耦**：同一套 Orchestrator 输出对接 Web 展示、ROS2 与人形仿真录屏。
5. **Isaac 双角色救援叙事**：Simple Room 家庭场景、固定机位、分阶段动画（躺—坐—站），避免实拍抠图穿帮。
